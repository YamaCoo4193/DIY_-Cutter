# AGENTS.md

このプロジェクトにおける AI エージェント（COMDEX / Cursor 等）向け行動指針。

---

## 🎯 プロジェクト目的

DIY木工向けの設計・材料算出・カット最適化 Webアプリを構築する。

初心者でも使える直感的UIと、
無駄を最小化する材料最適化ロジックを両立させること。

---

## 🧱 基本設計方針

- フロントエンド：TypeScript + React + Vite
- オブジェクト指向設計
- UI とドメインロジックを分離する
- ドメインロジックは React に依存しない
- 将来拡張（材料規格追加）を前提とする

---

## 🧠 AI 実装ルール

### 1. ロジック分離

- 計算ロジックを React コンポーネント内に直接書かない
- 計算は `domain/` または `services/` に実装する
- UI は状態管理と表示のみに専念する

---

### 2. 型安全性

- any を使用しない
- 明確な型定義を models に作成する
- 可能な限り interface よりも型安全な構造を使う

---

### 3. ドメイン優先

UI実装前に以下を優先する：

1. モデル定義
2. ドメインロジック
3. サービス層
4. UI接続

---

### 4. 拡張性

- 材料規格は配列定義で管理する
- 将来 1×6 / 2×6 を追加できる設計にする
- 刃厚（kerf）は設定可能にする

---

### 5. 禁止事項

- コンポーネント肥大化
- 1ファイルに大量ロジックを書く
- グローバル変数依存
- ハードコードされた材料値

---

## 🖥 UI ガイドライン

- モダンでおしゃれなデザイン
- 余白を活かす
- 選択中の部材はハイライト表示
- 材料規格ごとに色分け
- 視認性を重視

---

## 🧪 実装順序（重要）

必ず以下の順序で実装すること：

1. domain モデル
2. 材料算出ロジック
3. カット最適化
4. 設計画面 UI
5. 保存機能
6. デザイン改善

---

## 📦 データ保存

- v1 は LocalStorage
- JSON形式
- バージョン番号を持たせる
