# DIY Cutter 仕様書

## 1. 概要

DIY 木工向けの材料算出 Web アプリ。
ユーザーが必要部材を入力すると、材料規格ごとの必要本数、端材量、カット図を算出する。

現行実装は 1 画面構成の MVP で、対象機能は以下に限定する。

- 必要部材の手入力
- 材料規格ごとの使用可能な母材長さの選択
- 刃厚を考慮したカット計画の算出
- 結果の可視化
- LocalStorage への保存、再読込、削除

## 2. 対象ユーザー

- DIY 初心者から中級者
- 棚や天板などの部材切り出しを事前に整理したいユーザー
- ホームセンターで買う母材本数の目安をすぐ確認したいユーザー

## 3. 対応プラットフォーム

- PC ブラウザを主対象とした Web アプリ
- React + TypeScript + Vite で構築

## 4. 現行画面仕様

### 4.1 材料入力セクション

- 保存名入力
- 現在の入力内容を保存
- 刃厚 mm の数値入力
- 材料規格ごとの使用可能母材長さのチェックボックス
- 必要部材入力行の追加、更新、削除

各入力行で扱う項目:

- 材料種別
- 長さ mm
- 本数

未選択、0 以下、非数値の入力は計算対象から除外する。

### 4.2 保存済み結果セクション

- 保存済みスナップショットの一覧表示
- 保存名、保存日時、入力行数の表示
- 読み込み
- 削除

### 4.3 必要材料セクション

材料種別と母材長さの組み合わせごとに以下を表示する。

- 必要本数
- 母材長さ mm
- 端材が出る母材本数
- 端材長さの集計
- 補足メモ

選択した母材長さでは部材が収まらない場合、概算行を表示する。

### 4.4 カット図セクション

各母材ごとのカット計画を SVG で表示する。

- 母材全長
- 各切り出し部材長
- カット順
- 端材長

カット図が生成できない場合は代替メッセージを表示する。

### 4.5 部材結果セクション

材料種別ごとに以下を表示する。

- 入力された長さ別本数
- 合計本数
- 必要母材本数

## 5. 対応材料規格

現行実装で対応している規格:

- `1x4`
- `2x4`
- `cafe`

現行の母材長さ:

- `1x4`: `6f`, `8f`, `12f`
- `2x4`: `6f`, `8f`, `12f`
- `cafe`: `1000mm`, `2000mm`, `4000mm`

各規格は以下の属性を持つ。

- ID
- 表示名
- 幅 mm
- 厚み mm
- 表示色
- 選択可能な母材長さ一覧

## 6. 材料算出仕様

- 入力部材は材料種別ごとに集約する
- 本数分だけ切り出し部材へ展開する
- 刃厚は同一母材内で 2 本目以降の部材追加時に加算する
- 0 以下の刃厚は 0 として扱う
- 有効な部材のみで集計結果とカット計画を生成する

## 7. カット最適化仕様

現行実装は厳密最適化ではなく、軽量なヒューリスティックを採用している。

- 部材は長い順に処理する
- 既存母材に入る場合は端材が最小になる母材へ詰める
- 新規母材を開く場合は、その部材単体で端材が最少の母材長さを優先する
- 比較用として、母材長さ固定順のプランも複数生成する
- 候補プラン群から総端材量が最小の案を採用する
- 同一端材量なら母材本数が少ない案を優先する

選択された母材長さで収まらない部材がある場合、その材料種別の `cutPlans` は生成されないことがある。

## 8. 保存仕様

- 保存先は `localStorage`
- キーは `diy-cutter:material-estimates:v1`
- 保存形式は配列 JSON
- 各要素は `version` と `snapshot` を持つ
- 現行保存バージョンは `1`

保存内容:

- スナップショット ID
- 保存名
- 保存日時 ISO 文字列
- 刃厚
- 入力中の部材行一覧
- 材料規格ごとの母材長さ選択状態

壊れたデータやバージョン不一致データは、読込時に無視する。

## 9. 入力と表示のルール

- 部材長は mm 単位
- 本数は整数として扱い、小数は切り捨てる
- 保存名が空の場合は日時ベースの自動名を付与する
- 初期状態では空の入力行を 1 行表示する
- 初期母材選択は各材料規格の全長さを選択済みとする

## 10. スコープ外

現行実装には含まない。

- 図面エディタ
- ドラッグ操作によるレイアウト設計
- 3D 表示
- 価格計算
- クラウド保存
- 認証
